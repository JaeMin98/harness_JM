# Constitution

## 1. Purpose
이 저장소는 교육용 앱을 같은 방식으로 설계, 구현, 검증하기 위한 저장소다.
모든 결과물은 초보자가 읽고 따라갈 수 있어야 한다.

## 2. Core Values
- 간결성: 복잡하면 쪼개고, 불필요하면 제거한다.
- 검증 가능성: 문서, 코드, 테스트가 같은 결과를 검증할 수 있어야 한다.
- 안전성: 사용자 개인정보와 입력 데이터는 기본적으로 보호 대상이다.

## 3. Non-Negotiables
- 문서 없이 기능을 추가하지 않는다.
- 테스트를 통과시키기 위해 테스트를 속이지 않는다.
- 사용자 개인정보와 입력 데이터를 로그, 파일, DB, URL에 남기지 않는다.
- 한 번에 너무 많은 규칙을 넣지 않는다. 이해 가능한 수준으로 유지한다.

## 4. Document Order
문서가 충돌하면 아래 순서를 따른다.
1. Constitution
2. Repository security / reliability
3. App-level harness docs
4. Plans, notes, references

## 5. Roles
- PM: 범위를 정하고 완료 기준을 고정한다.
- Coder: 문서에 맞게 가장 단순한 구현을 만든다.
- Designer: 이해하기 쉬운 화면과 흐름을 설계한다.
- Security Reviewer: 데이터 유출 가능성을 막는다.
- Tester: 문서 기준으로 동작을 검증한다.

## 6. Definition of Done
- 목적과 범위가 문서에 적혀 있다.
- 구현이 해당 문서와 모순되지 않는다.
- 필요한 테스트와 검토를 통과한다.
- 다음 사람이 이어서 볼 수 있게 구조와 상태가 남아 있다.

## 7. Writing Style
- 짧게 쓴다.
- 한 문서에는 한 책임만 둔다.
- 예시는 작고 바로 실행 가능해야 한다.
- 초보자가 처음 읽어도 이해할 수 있는 표현을 쓴다.
