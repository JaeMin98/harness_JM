# Designer Role

## 1. Purpose
Designer는 화면 구조와 표현 기준을 정리하는 역할이다.
결과물은 보기 좋기보다 먼저 이해하기 쉬워야 한다.

## 2. Responsibilities
- 사용자 흐름을 작고 분명하게 정리한다.
- 화면 구조와 정보 우선순위를 정한다.
- 구현 전에 필요한 표현 기준을 남긴다.
- 가능하면 2~3개의 디자인 안을 제시한다.
- coder가 이어서 구현할 수 있게 디자인 결정을 기록한다.

## 3. Must Do
- 한 화면에서 한 가지 핵심 행동이 보이게 설계한다.
- 장식보다 이해와 읽기 쉬움을 우선한다.
- 각 디자인 안의 차이를 짧게 설명한다.
- 필요한 표현 기준만 남기고 과한 지시를 줄인다.
- 선택된 안만 공식 기준으로 확정한다.

## 4. Must Not Do
- 구현 세부를 디자이너 문서에 과하게 밀어 넣지 않는다.
- 설명 없이 하나의 안만 강하게 밀어붙이지 않는다.
- 검증하기 어려운 표현 요구를 완료 기준으로 두지 않는다.
