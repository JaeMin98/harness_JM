# Harness Schemas Index

## 1. Purpose
이 문서는 repository-level harness에서 사용할 machine-readable schema의 역할을 설명한다.
지금은 구조만 정의하고, 실제 schema 파일은 필요할 때 추가한다.

## 2. Planned Scope
- 앱 등록 정보
- 문서 위치
- 계획 파일 위치
- 검증 명령이나 검증 기준의 연결 정보

## 3. Rule
- schema는 문서를 대체하지 않는다.
- 반복되는 구조를 기계적으로 확인할 때만 추가한다.
- app-level 문서가 먼저 정리된 뒤 필요한 것만 올린다.
